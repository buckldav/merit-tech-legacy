What are the basic skills I'll need to know to program effectively?

  • Algorithms: How to solve the problem

    An algorithm is a step by step solution to a problem. Good programmers can take a large problem and break it down into smaller steps.

  • Commenting and Documentation

    Good programmers can communicate their logic and ideas well. To help yourself in organizing your code and to help future readers of your code understand what is going on, it's important to leave comments, which in python are lines starting with a #. An example of good commented code, where each idea is explained in a comment, can be seen below with a tic-tac-toe game.

    For bigger programs, it is useful to leave documentation, usually longer explanations of what code does and how it is organized presented as text, flowcharts, and other sorts of outlines.

  • Debugging: Dealing with errors

    Inevitably, there will be errors in your code. There are three types of errors in programming: syntax, runtime, and logic errors. As you become more experienced, the more apt you will be to deal with the errors that arise.

    1. Syntax Errors
      The equivalent of spelling and grammar errors in programming.
    2. Runtime Errors
      Errors that occur once the program is already running. Typically set off by a variable or input that the program is not equipped to handle.
    3. Logic Errors
      Your program runs fine, but it does something that you don't want. Perhaps you wanted two numbers to multiply together but you told the program to add them.

Style

Like written languages such as English, there are rules and conventions in programming languages to follow. These principles will make your code organized and readable.

Variables and Naming Conventions

When naming variables and constants, the most important thing is to BE CONSISTENT IN YOUR FORMATTING. If you use underscores in your names, use that naming convention throughout your entire program. Constants are variables in that they are like a box that holds data, but the data DOES NOT CHANGE. Constants should always be capitalized to distinguish them from variables. 

 
  num = 5     # Assigning the value 5 to the variable "num"
  camelCase   # Variable names cannot have spaces, so camelCase is a way to allow for multiple words in a variable name
  anotherExampleOfCamelCase
  under_score # If you don't like camelCase, use underscores but BE CONSISTENT IN YOUR NAMING.
  CONSTANT    # Always do ALL CAPS for constants.

Finally, make sure your variable names are descriptive. If you have a financial application with variables keeping track of dollars and cents, then name the variables dollars and cents respectively.

Types and Operators

Primitive Data Types

There are several different types of data that variables can store. Data types that are built in to python and are standard across most programming languages are called primitive data types. In python, these include: 


  integer = 0
  float = 0.0
  string = "words and letters"
  boolean = True or False

Integers are non-decimal numbers. Floats are decimal numbers. Strings are enclosed in quotes and can contain any number of letters, numbers, and characters. Finally, booleans are used for logical expressions and are either True (1) or False (0).

Functions

A function is a section of a program that performs a specific task. There are a few vocabulary words associated with functions:

  • Defining a function

    This is when you create the function. In python, this is done by using the def keyword.
    
  def myFunction(): # This line is defining the function "myFunction"

  • Scope

    This is what is within a function and outside a functions. We will talk about two levels of scope this semester, "local" and "global".
    Local is code only accessible inside the control structure (e.g. if/else) or function.
    Global is code accessible inside and outside the control structure or function.

    Example 1: Local variables with functions

    
  def myFunction():
    print("This code is local to myFunction.")
    print("This code is too, because it is indented at the same level as the above line.")
    print("Same with this one.")
    localVar = "Local"; # Scope of variable = local - only accessible within myFunction

  print("This code is not within myFunction because it is not indented on the same level like the above code.")
  print(localVar) # This does not work because localVar was declared within myFunction and is not accessible globally

    Example 2: Local variables with control structures

    
  def myFunction():
    condition = True
    if condition:
      localVar = 0
    print(localVar) # Doesn't work, because localVar is only accessible within the if statement above

Loops

Loops are ways to repeat code a certain number of times. There are several kinds of loops. The ones that you need to know are for loops and while loops (both pretest and posttest).

For Loops

For loops are great for two cases:

  1. When you need a loop to run a certain number of times.
  2. When you are parsing through a collection of items like an array or list.

While Loops

While loops behave similarly to conditionals (if statements) that they only execute when the condition is true.


  condition = False
  while condition:
    print("This code will never execute because the condition was false.")

When the condition is true, if statements only execute once. Loops, on the other hand, execute UNTIL the condition is false again.

When to use while loops

Whenever a for loop won't work for you. Cases can include:

  1. When you don't know how many times the loop will run. Rather, you want code to execute until a certain condition is false. 
    
  from random import randint
  shouldContinue = True

  while shouldContinue == True:
    print("running")
    rand = randint(1, 10)
    if rand > 5:
      shouldContinue = True
    else:
      shouldContinue = False
  2. When a loop just runs forever (until the program terminates). This is typical in game programming. 
    
  while True:
    print("Move the character")
    print("Check for collisions")
    # etc.
