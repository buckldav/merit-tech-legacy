What is an IDE?

An IDE is an integrated development environment. It's software that allows one to code efficiently, usually including tools like code syntax highlighting, an integrated terminal, and workspaces to help you organize your files. The IDE we are using in this class is Visual Studio and we will be programming in C#. In Visual Studio, hitting "Run" both compiles your code (converts it to machine-readable code) and runs it. The game engine we will be using in this class is Unity, which uses C# as the scripting language.

What are the basic skills I'll need to know to program effectively?

  • Algorithms: How to solve the problem

    An algorithm is a step by step solution to a problem. Good programmers can take a large problem and break it down into smaller steps.

  • Commenting and Documentation

    Good programmers can communicate their logic and ideas well. To help yourself in organizing your code and to help future readers of your code understand what is going on, it's important to leave comments, which in C# are lines starting with a // or /* for multiline comments */. An example of good commented code, where each idea is explained in a comment, can be seen below under "Operators".

    For bigger programs, it is useful to leave documentation, usually longer explanations of what code does and how it is organized presented as text, flowcharts, and other sorts of outlines.

  • Debugging: Dealing with errors

    Inevitably, there will be errors in your code. There are three types of errors in programming: syntax, runtime, and logic errors. As you become more experienced, the more apt you will be to deal with the errors that arise.

    1. Syntax Errors
      The equivalent of spelling and grammar errors in programming.
    2. Runtime Errors
      Errors that occur once the program is already running. Typically set off by a variable or input that the program is not equipped to handle.
    3. Logic Errors
      Your program runs fine, but it does something that you don't want. Perhaps you wanted two numbers to multiply together but you told the program to add them.

Functions / Methods

A function is a section of a program that performs a specific task. If a function is declared within a class, it is called a method. There are a few vocabulary words associated with functions:

  • Defining a function
    This is when you create the function. 
    
  void MyFunction() {
    Console.WriteLine("This function doesn't reutrn anything. It just prints this.");
  }
  • Scope
    This is what is within a function and outside a functions. We will talk about two levels of scope this semester, "local" and "global".
    Local is code only accessible inside the control structure (e.g. if/else) or function.
    Global is code accessible inside and outside the control structure or function.
    Example 1: Local variables with functions
    
  void MyFunction() {
    int number = 5;
  }
  // Doesn't work because "number" is local to MyFunction and not accessible outside of it.
  Console.WriteLine(number);
    Example 2: Local variables with control structures
    
  bool local = true;
  if (local) {
    string name = "George"
  }
  // Doesn't work because "name" is local within the {} of the if statement and not accessible outside of it.
  Console.WriteLine(name);
  • Calling a function
    This is when you use a function. Here is are some examples of calling functions with parameters and return values. 
    
  void MyFunction() {
    Console.WriteLine("George");
  }

  // Calling the function, will now print "George"
  MyFunction();

Loops

Loops are ways to repeat code a certain number of times. There are several kinds of loops. The ones that you need to know areforloops andwhileloops (both pretest and posttest).

For Loops

For loops are great for two cases:

  1. When you need a loop to run a certain number of times. 
    
  // Prints 0 through 9
  for (int i = 0; i < 10; i++) {
    Console.WriteLine(i);
  }
  2. When you are parsing through a collection of items like an array or list.
While Loops

While loops behave similarly to conditionals (if statements) that they only execute when the condition is true. 


  bool doTheLoop = false;
  while (doTheLoop) {
    // This part of the code is never reached
  }

When the condition is true, if statements only execute once. Loops, on the other hand, execute UNTIL the condition is false again. 


  bool doTheLoop = true;
  while (doTheLoop) {
    // Do some stuff

    // Reach some condition
    doTheLoop = false;
  }
When to use while loops

Whenever a for loop won't work for you. Cases can include:

  1. When you don't know how many times the loop will run. Rather, you want code to execute until a certain condition is false. 
    
  bool doTheLoop = true;
  while (doTheLoop) {
    // Do some stuff

    // Reach some condition
    doTheLoop = false;
  }
  2. When a loop just runs forever (until the program terminates). This is typical in game programming. 
    
  while (true) {
    // Check for inputs, update timer, etc.

    // This exits the while loop
    if (input == "quit") {
      break; 
    }
  }

Objects, Arrays, and Lists

Objects

Objects are containers that can hold several data members and functions. They are a way to represent physical objects in computer code.

All objects have:

  • Private data members (variables)
  • A constructor (a way to make an instance of an object)
  • Public getters and setters
  • Public methods (functions)

Here's an example of a Ball represented as an object:


  class Ball {
    // data members
    private int radius
    private float xVelocity
    private float yVelocity

    // constructor
    public Ball() {
      this.radius = 2
      xVelocity = 2
      yVelocity = 0
    }

    // getters and setters
    public int GetRadius()
    public float GetXVelocity()
    public float GetYVelocity()
    public void SetRadius(int radius)
    public void SetXVelocity(int radius)
    public void SetYVelocity(int radius)

    // methods
    public void Throw()
    public void Collide()
    public void Bounce()
  }

Arrays and Lists

Arrays and Lists are both collections that can hold several objects.

The key thing to know for now about arrays and lists are:

  • Arrays are static (fixed size). If you make an array of 5 numbers, you cannot add an additional number on the end.
  • Lists are dynamic (variable size). If you have a list of 10 names, you can add another name making the list have 11 names. Likewise, you may remove items from a list and it will resize accordingly.
  • Arrays and lists both start counting their indicies at 0. Example below:

  names = {"Boggis", "Bunce", "Bean"}
  names[0] = "Boggis"
  names[1] = "Bunce"
  names[2] = "Bean"
  names.Length = 3
