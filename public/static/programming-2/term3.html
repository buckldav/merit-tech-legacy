Input / Output

You need to know how to properly get input from the user and use their input to perform certain logical operations, finally returning output to them.

Input is typically done with string input = Console.ReadLine();

In C#, output is typically done with Console.WriteLine($"Output");
When outputting variables, we use String Interpolation which is done with a $ before the quotation marks. Example: 


  string food = "cheese";
  Console.WriteLine($"My favorite food is {word}.");
  
  Output: My favorite food is cheese.

Here's the C# tutorial we did towards the start of class which has a few examples of input and output.

NOTE: By default, Console.ReadLine(); reads in strings, even if the inputs are numbers.

Objects, Arrays, and Lists

NOTE: The test this term will only check your knowledge related to the concepts and ideas of the following, not any specific code syntax. We will cover that more in Term 4.

Objects

Objects are containers that can hold several data members and functions. They are a way to represent physical objects in computer code.

All objects have:

  • Private data members (variables)
  • A constructor (a way to make an instance of an object)
  • Public getters and setters
  • Public methods (functions)

Here's an example of a Ball represented as an object:


  class Ball {
    // data members
    private int radius
    private float xVelocity
    private float yVelocity

    // constructor
    public Ball() {
      this.radius = 2
      xVelocity = 2
      yVelocity = 0
    }

    // getters and setters
    public int GetRadius()
    public float GetXVelocity()
    public float GetYVelocity()
    public void SetRadius(int radius)
    public void SetXVelocity(int radius)
    public void SetYVelocity(int radius)

    // methods
    public void Throw()
    public void Collide()
    public void Bounce()
  }

Arrays and Lists

Arrays and Lists are both collections that can hold several objects.

The key thing to know for now about arrays and lists are:

  • Arrays are static (fixed size). If you make an array of 5 numbers, you cannot add an additional number on the end.
  • Lists are dynamic (variable size). If you have a list of 10 names, you can add another name making the list have 11 names. Likewise, you may remove items from a list and it will resize accordingly.
  • Arrays and lists both start counting their indicies at 0. Example below:

  names = {"Boggis", "Bunce", "Bean"}
  names[0] = "Boggis"
  names[1] = "Bunce"
  names[2] = "Bean"
  names.Length = 3
